Project

Andon & Production Management System

Ignition Perspective · Siemens PLC · Danfoss Tallahassee

🖱 Drag to pan · Scroll / Pinch to zoom · Tap ⊞ to fit

Challenge

The Danfoss Tallahassee plant required real-time visibility into production metrics and machine performance. Manual tracking of production counts, part locations, and downtime events was inefficient, and there was no systematic way to manage response times for critical support calls, hindering operational speed and accountability.

Solution

Developed a comprehensive SCADA solution using Ignition Perspective with direct PLC integration via standardized UDTs and handshakes. Key features include:

  • Interactive Monitoring: Dynamic home screen with color-coded station statuses (Green/Yellow/Red), part tracking (model/serial), and pan/zoom navigation.
  • Production Metrics: Real-time tracking of Shift/Hour goals, OK/Reject counts, FPY, and Pass Rate for specific production groups.
  • Cycle Time Analysis: Live monitoring vs. targets with visual progress alerts.
  • Escalation Pipeline: Robust Andon system with 5/15/35/90min automated email escalation from Supervision to Directorate levels.
  • Traceability: Searchable production and call history with Active Directory security and Excel export capabilities.

Technologies Used

Ignition PerspectiveSiemens PLCSQLOPC-UASepasoft

Outcomes

Reduced response times to line stoppages through immediate visibility and automated escalation. The system provided deep data-driven accountability and was so successful that it was replicated across all three major production lines in the factory.

Project

Global Software Standardization

Siemens S7-1500 · Ignition SCADA · MES (Siemens SIT) · Danfoss USA

Challenge

The Turbocor compressor assembly line in Tallahassee, FL, utilized original manufacturer (OEM) logic that deviated from Danfoss's global "Manufacturing Excellence" standards. The system required a complex migration to a more passive PLC role, where the corporate MES would orchestrate the production flow.

Solution

Working with a senior team, I executed the refactoring of both the PLC and SCADA layers during a 10-month project, culminating in a 2-month on-site commissioning phase in the USA. Key technical pillars included:

  • Ignition as Middleware: Positioned Ignition as the bridge between the Siemens Opcenter MES and the PLC, managing XML-based telegram exchanges for real-time production orchestration.
  • Dynamic Flow Control (SFC): Integrated the system with Sequential Function Charts in Ignition to manage automatic workflows, Electronic Work Instructions (EWI), and result logging.
  • Standardized PLC Architecture: Rebuilt the Siemens S7-1500 code using modular Function Blocks (FBs) to integrate Cognex Vision, Atlas Copco Nut Runners, RFID, and many other devices via Profinet.
  • Unified UI/UX: Developed standardized station interfaces with integrated EWI, visual I/O mapping, and Active Directory authentication for role-based security.
  • Rigorous Validation: Executed complex FAT and SAT procedures, ensuring absolute alignment with global Danfoss standards for high-performance manufacturing.

Technologies Used

Siemens S7-1500IgnitionSiemens Opcenter MESProfinetSQL

Outcomes

Successfully migrated the entire assembly line to the new global standard. The solution provides a scalable, maintainable codebase that ensures full repeatability and production traceability, directly validated through successful 24/7 operations in the Florida plant.

OEE & Downtime Management Solution

Ignition · Sepasoft MES · Rockwell Automation · Lear Caçapava

Challenge

Lear Automotive struggled with inaccurate manual downtime logs and untracked micro-stoppages, making it impossible to calculate a reliable OEE baseline. The plant needed a system that enforced data integrity and provided granular visibility into station-level cycle times.

Solution

As the Technical Lead, I designed and deployed a robust MES solution using Ignition and the Sepasoft OEE module, integrated with Rockwell PLCs via handshake logic:

  • Data Integrity via Interlocking: Implemented a "No-Justification, No-Production" logic where the PLC locks the machine if a downtime event isn't coded or a support call exceeds a 5-minute threshold.
  • Three-Way Support Workflow: Developed a secure 3-step call system (Open, Accept, Close) with RFID authentication to ensure accountability between Production, Maintenance, and Quality.
  • Granular Tracking: Real-time cycle time charts (Mean & Standard Deviation) for every line station and a dedicated Production Mix Indicator to monitor seat model distribution.
  • Resilient Architecture: Integrated contingency bypass "Test Modes" to ensure factory operations continue during IT network maintenance or failures.
  • Management Controls: Developed secure interfaces for shift management and planned stop scheduling.

Achieved 100% data reliability through machine interlocking across 4 major lines (Cushion, Press, Welding, and Assembly). The real-time OEE visibility allowed management to eliminate untracked micro-stoppages and optimize the production mix effectively.

Project

Java Open Protocol Module for Ignition

Java · Ignition Module SDK · Open Protocol

Challenge

Industrial tightening tools from Atlas Copco communicate via Open Protocol (TCP/IP), which is not natively supported by Ignition. There was a clear market gap for a robust, native integration that could handle complex fastening data within the SCADA/MES loop without overhead.

Solution

Developed a fully functional custom module using the Ignition Module SDK and Java. The product abstracts protocol complexity into a scalable, multi-threaded architecture with a clear separation of responsibilities:

  • Scalable Architecture: Built on strict OOP principles, utilizing a centralized registry to map MIDs to specific parsers (e.g., Mid0061Parser.java) for structured payload decoding.
  • Intelligent Threading: Optimized Gateway performance using dedicated Java threads for network listening and Ignition's ExecutionManager for scheduled tasks like keep-alive (MID 9999).
  • Managed Tag Provider: High-performance data exposure via a native "Open protocol" Tag Provider with dynamic hierarchical structures for identification, subscriptions, and requested data.
  • Synchronous RPC API: Python scripting functions (system.openprotocol.*) using CompletableFuture to allow scripts to wait for tool confirmation without blocking system processes.
  • Controller Simulator: A multi-threaded Python simulator used as a "digital twin" to validate the module offline, including proactive event simulation (MID 0061).

Technologies Used

JavaIgnition SDKOpen ProtocolTCP/IPPythonMultithreading

Outcomes

The module fills a major integration gap, providing full production traceability. It was successfully tested and validated at the DAF factory in Brazil, demonstrating absolute stability in managing fastening data and tool states for high-volume assembly lines.

Field Service

CANopen Network Interface – Posigraf

CANopen · B&R · Custom Microcontroller

Situation

Posigraf's printing press used a B&R industrial PC as the motion master, communicating with a fleet of drives over CANopen. The drives needed replacement but the new models had a different CANopen profile (PDO mapping), making them incompatible with the existing B&R program — which could not be modified.

Intervention

Designed and built a custom microcontroller-based adapter that sat between the B&R master and the new drives. The adapter presented the old CANopen profile to the B&R while translating all commands and status words to the new drive's profile in real time — effectively masking the hardware change from the host controller.

Technologies Used

CANopenB&R Automation PCMicrocontroller (ATmega328P)Reverse Engineering

Result

Full machine compatibility restored without touching the B&R source code. Machine resumed operation with zero visible change to operator or existing software.
